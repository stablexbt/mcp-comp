# Contributing to REM Payment Server

Thank you for your interest in contributing to REM Payment Server! This guide will help you get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/rem-payment-server.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Run tests: `npm test`
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature`
8. Open a Pull Request

## Development Setup

```bash
# Install dependencies
npm install

# Run in development mode
npm run dev

# Run tests
npm test

# Run linting
npm run lint
```

## Code Style

- Use TypeScript for all new code
- Follow the existing code style (ESLint config provided)
- Write tests for new features
- Update documentation when adding new API endpoints

## Pull Request Guidelines

- Keep PRs focused on a single change
- Include tests for new functionality
- Update the README if adding new features
- Reference related issues in your PR description

## Reporting Issues

- Use the GitHub issue templates
- Include steps to reproduce bugs
- Include your environment details (Node.js version, OS, etc.)
